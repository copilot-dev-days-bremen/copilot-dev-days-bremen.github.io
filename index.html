What you will learn

GitHub Copilot Dev Days is an introductory, hands-on event for developers who want to understand how AI can support modern software development. Participants will learn what GitHub Copilot is, how it works, and how it can be used as a practical coding companion in everyday development tasks—without requiring prior AI knowledge. The event is aimed at developers who are new to GitHub Copilot and AI-assisted coding and want a clear, practical starting point.

We have many workshops and sessions planned to help you get the most out of GitHub Copilot. You can learn the following fundamentals:

Understand what GitHub Copilot is and where it adds value in daily development

Learn what GitHub Copilot is, how it works under the hood, and in which development scenarios it provides real, practical value.

Learn practical use cases and limitations of AI-assisted coding

Discover how AI can support everyday development tasks such as writing code, understanding existing codebases, exploring unfamiliar APIs, refactoring, and testing—while knowing where its limits are.

Gain confidence to start using Copilot responsibly in real projects

Gain the confidence to start using GitHub Copilot in real projects, with a solid understanding of best practices, quality considerations, and how to stay in control as a developer.

Sessions and Talks

Listen to experts

Although we have many workshops prepared for you, these sessions will provide additional insights and learning opportunities.

Session

Beginner

Github Copilot Overview: What is this all about and what does it include?

This session provides a concise overview of GitHub Copilot, explaining what it is, how it works, and what capabilities it includes today. We’ll look at how Copilot supports developers across the software lifecycle—from writing and understanding code to testing and documentation—while clarifying where it adds real value and where human judgment remains essential. The goal is to demystify the tool, set realistic expectations, and give you a clear picture of how GitHub Copilot fits into modern development workflows.

Sven Sieverding

Session

Beginner

Spec Driven Development

Spec Driven Development turns specifications into the true starting point of software development. In this session, you’ll see how clear, well-structured specs—requirements, examples, and acceptance criteria—can be used directly by GitHub Copilot to generate code, tests, and refactorings. Learn how shifting from code-first to intent-first development leads to better results, fewer misunderstandings, and more predictable software—with Copilot as a powerful implementation partner rather than just a code autocomplete.

Jörn Syrbe
